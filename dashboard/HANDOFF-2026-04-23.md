## Addendum 2 — dark mode, toast, mobile, a11y (same night)

Still zero migrations. Back-to-back front-end pass that finishes every punted item from the earlier "Not yet done" list.

### Dark mode

Class-based Tailwind dark mode (`darkMode: 'class'`). Three-state theme cycler (light → dark → system) persisted in `localStorage['purity-theme']`. Pre-hydration inline script sets `<html class="dark">` before first paint to avoid a light-flash. `prefers-color-scheme` fallback when preference is unset.

### Toast

`ToastProvider` + `useToast()` hook. 3 s auto-dismiss, stack limit 4, newest first, bottom-right on ≥ sm and full-width bottom strip on mobile. Success / error / info kinds. Replaces every inline "Saved ✓" / "Error" span across the app.

### Mobile responsiveness (375 px baseline)

Nav is now horizontally scrollable at narrow widths. Chat grid collapses at `lg:` instead of `md:`. All data tables wrap in `overflow-x-auto` with sensible `min-w-*` so horizontal scroll activates cleanly on phones instead of squishing columns. ManualUpdateButton collapses to an `↻` icon at `< sm`.

### Accessibility

Skip-to-main anchor at top of `<body>`. `:focus-visible` aqua ring on every interactive (keyboard-only, not on mouse click). Every icon-only button has an `aria-label`; every decorative emoji is wrapped in `<span aria-hidden>`. All previously label-less bibliography filter controls got `aria-label`. `role="search"` + `aria-label` on the chat + semantic-search forms. `aria-live="polite"` on dynamic status regions (label confirmation, toasts).

### URL persistence on bibliography filters

`FormAutoSubmit` client helper: any `change` event on the filter form (except on the debounced title input) triggers `form.requestSubmit()`, so topic / category / rights / year / pdf_only / open_only all update the URL immediately without clicking Apply.
